# 下行飞行控制协议

服务器向设备发布到：`{namespace}/{vendor_id}/control/set`，QoS 2。

设备将命令成功写入 STM32 串口后，立即发布一次“执行中”回执；收到
单片机 `COMMAND_ACK` 后再发布最终回执。两类回执均发布到：
`{namespace}/{vendor_id}/control/ack`，QoS 2。

成功下发后的立即回执示例：

```json
{"command_id":"flight-001","command":"takeoff","mavlink_command":31091,"status":"in_progress","result_code":"pending","message":"命令已下发，正在等待单片机应答"}
```

固件回复 `MAV_RESULT_ACCEPTED` 后的最终回执示例：

```json
{"command_id":"flight-001","command":"takeoff","mavlink_command":31091,"status":"accepted","result":0,"result_code":"accepted","message":"单片机已执行命令"}
```

`accepted` 的业务含义为“完成”。固件拒绝或执行失败时仍返回 `rejected`，
等待固件回复超时时返回 `timeout`。串口不可用、STM32 身份未学习或
串口写入失败时，不发布 `in_progress`。

双方职责约定如下：

- 树莓派负责在命令成功写入 STM32 串口后代固件发布
  `status=in_progress`、`result_code=pending`，固件无需为命令刚开始执行
  重复发送该状态。
- 固件负责在动作完全结束后发送最终 `COMMAND_ACK`。动作成功结束时返回
  `MAV_RESULT_ACCEPTED`；拒绝、中途失败或取消时返回对应的 MAVLink 最终结果。
- 因为最终 ACK 的发送时机由固件绑定到实际动作结束，所以树莓派收到
  `MAV_RESULT_ACCEPTED` 后发布的 `accepted` 能够表示真实完成，而非仅表示
  命令已受理。

## 服务器命令

```json
{"command_id":"pwm-001","command":"set_motor_pwm","parameters":{"pwm_us":[1650,1650,1570,1530]}}
```

```json
{"command_id":"flight-001","command":"takeoff","parameters":{}}
```

`command` 还支持 `land` 和 `emergency_stop`。四路 PWM 均为实际高电平脉宽，单位 `us`，合法范围 `1000..2000`。

## MAVLink 映射

| command | COMMAND_LONG id | param1..4 |
|---|---:|---|
| `set_motor_pwm` | 31013 | 电机 1..4 PWM 脉宽，单位 us |
| `emergency_stop` | 31090 | 未使用 |
| `takeoff` | 31091 | 未使用 |
| `land` | 31092 | 未使用 |

起飞命令执行前，单片机检查 IMU、气压环境模块和姿态数据新鲜度。检查失败时返回临时拒绝，并产生消息日志 `message_id=32`：`姿态或环境异常，一键起飞失败`。起飞过程中条件异常时同样产生该日志，并自动切换到降落过程。

树莓派上报该日志时会附加：

```json
{"event":"takeoff_sensor_failure","level":"warning","message":"姿态或环境异常，一键起飞失败"}
```
