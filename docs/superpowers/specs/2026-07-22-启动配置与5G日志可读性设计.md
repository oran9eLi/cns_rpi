# 启动配置与 5G 日志可读性设计

**日期：** 2026-07-22  
**状态：** 部分实施（C++ 主程序启动配置摘要已完成，Python 5G 守护进程待实施）

## 1. 目标

在不增加日志洪泛和不暴露凭据的前提下，提高树莓派两个常驻服务的现场可读性：

- `cns-rpi.service` 启动时输出主程序实际生效的配置摘要；
- `cellular-dialup.service` 启动时输出 5G 守护进程实际生效的配置摘要；
- Python 5G 守护进程的日志格式、时间语义和级别过滤与 C++ `logging::Logger` 保持一致；
- 长时间稳定运行时以低频健康摘要证明检测循环仍在工作；
- 两个进程不并发写入同一个有界滚动文件。

本轮不修改拨号、链路探测、链路恢复、MQTT 重连或 MAVLink 通信逻辑。

## 2. 方案选择

采用“Python 轻量日志器 + journald”方案：

1. Python 侧实现与 C++ 日志器相同的单行格式和四级过滤；
2. 5G 守护进程遵循全局 `logging.level`；
3. 5G 日志始终输出到 stdout/stderr，由 journald 收集，不使用 `logging.file`；
4. systemd 将 5G 进程的日志标识设为 `cellular_dialup`，主程序保持 `cns_rpi`。

不选择外部 `logger` 命令，以免增加子进程开销和测试依赖；不引入跨语言日志服务，因为超出当前问题的必要复杂度。

## 3. 统一日志格式

Python 侧每条记录使用与 C++ 一致的格式：

```text
[2026-07-22 14:41:07.123] [+000900.521s] [WARN ] 5G链路状态变更: DEGRADED
```

- 系统时间可信时使用 `YYYY-MM-DD HH:MM:SS.mmm`；
- 系统时间无效时使用既有占位 `---------- --:--:--.---`；
- 运行时长从当前进程启动时计算，使用单调时钟；
- 级别固定为 `DEBUG`/`INFO `/`WARN `/`ERROR`；
- 消息中的 CR/LF 替换为空格，保证一条业务日志只占一行；
- `INFO` 及以下写 stdout，`WARN` 和 `ERROR` 写 stderr；
- `logging.level` 同时控制 C++ 和 Python 进程的最低输出级别。

journald 外层保留主机、服务标识和 PID，联合查看时可区分来源：

```text
Jul 22 14:41:07 raspberrypi cellular_dialup[22094]: [2026-07-22 14:41:07.123] [+000900.521s] [WARN ] 5G链路状态变更: DEGRADED
Jul 22 14:42:14 raspberrypi cns_rpi[22291]: [2026-07-22 14:42:14.270] [+000960.427s] [INFO ] MQTT连接断开
```

## 4. 启动配置摘要

### 4.1 C++ 主程序

日志器创建成功后，主程序以 `INFO` 输出单行中文运行参数摘要，只覆盖可通过远程配置命令修改的四个白名单参数：

```text
当前运行参数：遥测上报周期=1000毫秒，心跳发送周期=5000毫秒，MQTT重连初始等待=1秒，MQTT重连最大等待=30秒
```

不输出配置文件路径、串口、MQTT连接信息、topic、凭据、学校名称、5G状态路径或已应用请求号列表，避免启动日志冗长。

任何日志级别都不输出原始遥测 JSON。MQTT 发布失败仍输出 `WARN` 事件日志。

### 4.2 Python 5G 守护进程

配置读取和日志器创建成功后，守护进程以 `INFO` 输出单行配置摘要，覆盖：

- 配置文件路径和日志级别；
- APN、CID、AT USB interface number 和 AT 枚举等待时间；
- 网卡名称和探测目标；
- 探测周期、离线/在线迟滞阈值和信号采样周期；
- 重拨后重置阈值、恢复退避初值和上限；
- 快照文件路径。

APN 为运营商网络选择参数，当前配置不包含 SIM PIN 或拨号密码，可在启动摘要中显示。未来如增加凭据字段，必须默认脱敏。

## 5. 5G 运行期日志

### 5.1 事件日志

- 服务启动、找到 AT 口、拨号成功和进入 `ONLINE` 记为 `INFO`；
- 进入 `DEGRADED`、AT 口未发现、质量查询失败和单次拨号失败记为 `WARN`；
- 进入 `OFFLINE`/`RECOVERING`、连续重拨失败或模块软重启失败记为 `ERROR`；
- 运行中重复的“未发现 AT 口”保留现有恢复退避节奏，不额外高频输出；
- 状态未变化时不重复打印状态事件。

### 5.2 周期健康摘要

守护进程每 5 分钟输出一条 `INFO` 单行健康摘要，不增加新配置项：

```text
5G链路健康: state=ONLINE network=NR5G-SA rsrp=-91dBm rsrq=-11dB sinr=18dB ip=100.73.167.140 tx=123456B rx=654321B
```

- 无线指标未采到时使用 `-`，不用 `null`；
- 摘要反映最近一次已采集快照，不额外触发 AT 查询或网络探测；
- 当 `logging.level` 为 `warn` 或 `error` 时，周期摘要被正常过滤；
- 链路不在线时仍可输出当前状态，但立即故障事件已由 `WARN`/`ERROR` 承担，健康摘要不替代故障日志。

## 6. 查看方式

联合跟踪两个服务：

```bash
journalctl -u cns-rpi.service -u cellular-dialup.service -f
```

只查看应用生成的统一日志行，隐藏 journald 外层元数据：

```bash
journalctl -u cns-rpi.service -u cellular-dialup.service -f -o cat
```

## 7. 故障处理

- 在日志器建立之前发生的参数错误或 JSON 读取错误仍使用 stderr 输出，确保启动失败可见；
- `logging.level` 非法时，Python 服务与 C++ 主程序一样拒绝启动并明确报错；
- 健康摘要某个字段缺失不得导致守护进程退出；
- 日志格式化不得反向影响链路状态机或快照写入。

## 8. 测试与验收

### 8.1 自动化测试

- Python 日志器覆盖四级过滤、标准流选择、可信/无效时间、单调运行时长和换行清理；
- 启动运行参数摘要严格覆盖四个远程可配置白名单参数，且使用中文名称和单位；
- 主程序和日志模块中不存在原始遥测 JSON 输出入口；
- 5G 健康摘要覆盖在线值、缺失值和 5 分钟限频；
- 既有拨号、链路状态机、快照和 C++ 日志测试全部回归；
- systemd 合约测试确认 `SyslogIdentifier=cellular_dialup`。

### 8.2 树莓派验收

5G 模块当前已拔出。实机验收前必须先提醒操作者插回模块，然后验证：

1. 主程序启动时只输出四个中文运行参数，且任何级别均不输出原始遥测 JSON；
2. journald 中进程标识分别为 `cns_rpi` 和 `cellular_dialup`；
3. Python 日志的时间、运行时长和级别格式与 C++ 一致；
4. 模块在线时低频输出健康摘要，拔插过程仍可看到即时状态和恢复事件；
5. `journalctl -o cat` 可联合连续查看两个服务的统一日志行。
