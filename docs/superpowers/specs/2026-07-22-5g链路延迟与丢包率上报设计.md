# 5G 链路延迟与丢包率上报设计

## 目标

在现有 `telemetry.cellular_5g` 中补充设备经 5G 链路到 CNS 公网弹性 IP
`112.124.52.232` 的延迟和丢包率。探测必须绑定 `usb0`，不得被 Wi-Fi
链路替代。

## 方案

- `cellular-dialup.service` 每 5 秒向 `112.124.52.232` 发送 1 个 ICMP
  探测包，并保留最近 12 次结果（约 60 秒滑动窗口）。
- `latency_ms` 为窗口内成功样本的平均 RTT，保留 1 位小数。
- `packet_loss_percent` 为窗口内失败样本占比，范围为 `0.0～100.0`，
  保留 1 位小数。
- 启动后按已有样本立即计算，不上报样本数；尚无探测结果时两项均为
  `null`。窗口内全部失败时，延迟为 `null`、丢包率为 `100.0`。
- 新增 `cellular.quality_probe_interval_seconds`，默认值为 `5`；旧配置
  缺少该字段时使用默认值。

质量采样独立于现有 10 秒双目标链路判定。`119.29.29.29` 继续只用于辅助
判断公网可用性；质量探测失败不单独修改 `last_error`、不直接触发恢复，也不
改变发给 STM32 的 `RPICELL` 语义。每次质量采样后刷新状态快照。

## 遥测字段

```json
"cellular_5g": {
  "latency_ms": 42.7,
  "packet_loss_percent": 8.3
}
```

快照读取端必须兼容字段缺失和 `null`，确保 Python 守护服务与 C++ 主程序可
分步升级。

## 验证

自动测试覆盖 ICMP RTT 解析、混合成功/失败窗口、全丢包、窗口淘汰、快照与
MQTT 序列化以及旧快照兼容。树莓派实机验证探测绑定 `usb0`、5 秒更新周期和
模块拔插后的统计变化；拔插操作仅在用户明确指示后执行。
