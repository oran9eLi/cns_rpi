# 独立日志模块设计

版本：2026-07-18  
状态：已确认，待实施

## 1. 目标

把生产程序的日志职责从 `src/main.cpp` 和 `src/mqtt/mqtt_client.cpp` 中拆出，集中到独立的 `logging` 模块，并接通 `config.json` 中的日志级别和文件输出配置。

本次同时满足并收紧 `docs/M7系统化部署设计.md` 第 2 节的验收要求：任何级别都不逐帧打印 MAVLink 字段；只有 `debug` 模式记录实际成功提交给 MQTT 的紧凑遥测 JSON。

## 2. 范围

本次接入：

- `src/main.cpp` 的启动、运行状态、可恢复错误和实际发送的遥测 JSON 日志。
- `src/mqtt/mqtt_client.cpp` 的连接、断开、订阅失败和队列溢出日志。
- `config.json` 的 `logging.level`、`logging.file` 和单文件容量配置。

本次不接入：

- `cns_control_test` 命令行工具。该工具的 stdout JSON 是机器可解析接口，stderr 是人工诊断接口，不得混入生产日志格式。
- 多文件轮转、压缩归档、网络日志和远程日志上传。
- 远程配置命令对白名单的扩展。日志配置只在进程启动时读取，修改后通过既有重启流程生效。
- OverlayFS 的挂载和权限切换。它仍属于 M7 后续工作。

## 3. 配置契约

配置格式调整为：

```json
"logging": {
  "level": "info",
  "file": "",
  "max_file_size_kb": 1024
}
```

字段规则：

| 字段 | 规则 |
|---|---|
| `logging.level` | 只接受 `debug`、`info`、`warn`、`error`，区分大小写 |
| `logging.file` | 空字符串表示控制台模式；非空表示文件模式 |
| `logging.max_file_size_kb` | `64..102400`，默认和示例值为 `1024`，即 1 MiB |

`max_file_size_kb` 是启动配置，不是 MQTT 运行时配置命令的可修改字段。配置字段缺失、类型错误或越界时，`LoadAppConfig()` 返回配置错误，程序不带错误配置继续启动。

## 4. 模块边界

新增：

```text
src/logging/logger.hpp
src/logging/logger.cpp
```

`logging::Logger` 负责：

- 日志级别过滤。
- 时间戳和级别前缀格式化。
- 控制台 stdout/stderr 分流。
- 文件创建、追加、容量检查和单文件向上滚动。
- 多线程写入串行化。
- 文件输出失败后的 stderr 降级。

日志模块额外提供遥测发送调试入口：

```cpp
void LogPublishedTelemetry(Logger& logger, std::string_view compact_json);
```

该函数只接收已经提交给 MQTT 的紧凑 JSON 字符串，不参与状态读取、JSON 构造、MAVLink 解码或状态更新。原有逐帧 `LogTelemetry()` 和 `LogJsonPayload()` 删除，不迁移到新模块。`main.cpp` 只保留组合、业务调度和日志调用。

`mqtt::MqttClient::Open()` 接收一个不拥有所有权的 Logger 引用。Logger 生命周期由 `main()` 管理，并覆盖 MQTT 客户端及其回调线程的完整生命周期。MQTT 模块不再直接写 `std::cout` 或 `std::cerr`。

## 5. 日志级别

级别从低到高为：

```text
debug < info < warn < error
```

配置级别表示允许输出的最低级别。

分类规则：

- `debug`：实际成功提交给 MQTT 的紧凑遥测 JSON。
- `info`：程序启动、MQTT 连接与断开、开始连接 broker、服务状态变化。
- `warn`：可恢复失败、消息被忽略、队列溢出、发布失败后等待重试。
- `error`：配置读取失败、串口打开失败等导致程序无法继续的错误。

单片机每来一帧只更新 `StateStore`，不得产生日志。遥测 JSON 只在达到 `runtime.telemetry_publish_interval_ms` 周期时构造一次；MQTT `Publish()` 成功接受该 payload 后，`debug` 模式调用 `LogPublishedTelemetry()` 记录同一份字符串。发布失败只记录一条 `warn`，不得再次打印 JSON。

## 6. 输出格式与目标

每条日志占一行，并包含固定宽度的本地系统时间、进程启动后的单调时间和级别：

```text
[2026-07-18 13:20:15.123] [+000123.456s] [INFO ] MQTT已连接
[---------- --:--:--.---] [+000003.218s] [WARN ] MQTT连接失败
```

系统时间早于 `2025-01-01 00:00:00` 时视为无效，使用固定 23 字符的 `---------- --:--:--.---` 占位；运行中 RTC 或 NTP 把时间校准到可信范围后，后续日志自动使用真实时间，既有记录不回填。单调时间从本进程启动时归零，用于绝对时间无效或运行中跳变时判断本次启动内的事件顺序。

级别字段固定 5 个字符，`INFO`、`WARN` 等较短名称补空格。普通消息中的换行符转换为空格，保证一条记录不会被拆成多行，便于单文件按完整记录滚动。

遥测 JSON 使用 `nlohmann::json::dump()` 的紧凑单行结果，不使用 `dump(2)`，不对 JSON 插入额外空白或换行：

```text
[2026-07-18 13:20:15.123] [+000123.456s] [DEBUG] telemetry={"vendor_id":"DCDWCNS1...","battery":{...}}
```

控制台模式：

- `debug`、`info` 输出到 stdout。
- `warn`、`error` 输出到 stderr。

文件模式：

- 所有允许级别写入 `logging.file` 指定的同一个文件。
- 不重复输出到 stdout 或 stderr。
- 文件打开失败时，Logger 初始化失败并让程序以明确错误退出。
- 运行过程中追加、滚动或同步失败时，Logger 切换到 stderr，并只报告一次文件输出故障，避免递归刷屏；主业务继续运行。

## 7. 单文件向上滚动

只保留 `logging.file` 指定的一个文件，不产生 `.1`、`.2` 等历史文件。

写入规则：

1. 写入前计算本条完整日志记录的字节数。
2. 若本条记录大于整个容量上限，则把消息截断为可容纳长度，并保留时间和级别前缀。
3. 若追加后不超过上限，直接追加。
4. 若追加后会超过上限，从现有日志中删除最旧约 25% 的内容。
5. 删除位置向后移动到下一条完整记录的开头，避免保留半条日志。
6. 把保留内容和新日志写入同目录临时文件，完成 `fsync` 后原子替换原文件，再对目录执行 `fsync`。
7. 轮转后文件不得超过配置上限，记录顺序始终从旧到新。

触发一次滚动后通常保留最新约 75% 内容，避免每写一条日志都搬移整个文件。

## 8. 并发和生命周期

Logger 内部使用互斥锁保护级别判断后的实际输出、文件容量状态和滚动过程。

主循环和 libmosquitto 回调线程可以同时调用 Logger，但一次只写入一条完整记录。Logger 不启动独立后台线程、不维护无界队列，进程退出时不需要额外等待日志线程。

Logger 不复制 MQTT 密码等敏感配置。日志调用方不得打印 `mqtt.auth.password`。

## 9. 错误处理

- 配置级别或容量非法：配置加载失败。
- 文件路径为空：进入控制台模式，不视为错误。
- 文件路径指向目录、父目录不存在或无权限：Logger 初始化失败，程序退出并向 stderr 给出原因。
- 启动时已有日志超过上限：在第一条新日志写入前执行一次向上滚动。
- 运行时文件写入失败：降级到 stderr，主程序继续运行。
- 系统时间无效或格式化失败：使用 `---------- --:--:--.---`，仍输出单调时间、级别和原始消息。

## 10. 测试与验收

自动化测试至少覆盖：

1. 四个合法级别的解析和非法级别拒绝。
2. 不同最低级别下的过滤结果。
3. 控制台模式的 stdout/stderr 分流。
4. 文件模式不重复写控制台。
5. 日志行的真实时间、无效时间占位、单调时间和级别均固定宽度，消息换行不会拆分记录。
6. 文件达到 1 MiB 后仍只有一个文件。
7. 滚动删除最旧记录、保留最新记录，且不保留半条记录。
8. 单条超大日志不会突破容量上限。
9. 启动时已有超限文件能在下一次写入时收敛。
10. 文件初始化失败和运行时写入失败的错误语义。
11. 任意级别均不输出逐帧 MAVLink 字段；`debug` 只在遥测 MQTT 发布成功后输出同一份紧凑单行 JSON，`info` 不输出遥测 JSON。
12. MQTT 回调和主线程并发写入不会产生交错或损坏记录。
13. `cns_control_test` 的 stdout JSON 和 stderr 行为不变。

验收时在树莓派上分别使用控制台模式和文件模式启动服务，确认 journald 能收到控制台日志，文件模式容量不超过 1 MiB，且无 SIM 卡导致的拨号重试不影响主程序日志模块运行。
