# 5G 单目标复用探测实施计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** 让 5 秒质量探测同时为 10 秒链路状态机提供结果，并通过迟滞消除单次丢包造成的状态抖动。

**Architecture:** 质量探测只访问配置中第一个目标，默认是 CNS 弹性公网 IP。守护服务保存最近一次探测是否成功；状态周期只消费该结果，不再发送第二次 ICMP。

**Tech Stack:** Python 3、unittest、Linux ping。

## Global Constraints

- `ONLINE` 下单次失败保持在线，连续 2 次状态判定失败才降级，连续 3 次离线。
- `DEGRADED` 后连续 2 次状态判定成功恢复在线。
- 5 秒质量统计和 STM32 `RPICELL` 语义保持不变。

### Task 1: 状态机迟滞

**Files:** `scripts/cellular_link.py`、`tests/test_cellular_link.py`

- [ ] 先写单次失败保持在线、第二次降级、第三次离线和成功清零的失败测试。
- [ ] 运行 `python3 -m unittest tests.test_cellular_link -v` 确认失败。
- [ ] 为 `LinkStateMachine` 增加降级阈值并实现最小逻辑。
- [ ] 重跑测试确认通过。

### Task 2: 复用质量探测

**Files:** `scripts/cellular_dialup.py`、`tests/test_cellular_dialup.py`、`scripts/cellular_link.py`、`config/config.example.json`

- [ ] 先写 0、5、10 秒只发生 3 次质量探测、状态探测器不再存在且状态周期复用最近结果的失败测试。
- [ ] 确认失败后删除守护进程的独立状态 ICMP，保存并消费最近质量结果。
- [ ] 默认目标和示例配置改为只含 `112.124.52.232`；旧配置多目标只取第一个。
- [ ] 运行全部 Python 测试确认通过。

### Task 3: 全量验证与部署

- [ ] 运行 `python3 -m py_compile scripts/cellular_link.py scripts/cellular_dialup.py`。
- [ ] 运行 `cmake --build build -j2 && ctest --test-dir build --output-on-failure && git diff --check`。
- [ ] 提交、推送并在树莓派执行 `scripts/deploy.sh`，核对服务状态和日志。
