# 独立日志模块实施计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 把生产日志从 `main.cpp` 和 `mqtt_client.cpp` 迁入独立模块，接通级别与 1 MiB 单文件向上滚动，并保证 debug 只记录实际成功提交给 MQTT 的紧凑遥测 JSON。

**Architecture:** `logging::Logger` 是同步、线程安全、无后台队列的日志边界；控制台模式按级别分流 stdout/stderr，文件模式在同一文件达到上限时原子保留最新记录。`main()` 拥有 Logger，MQTT 客户端只保存非拥有型指针，遥测发送成功后复用实际 payload 写一条 debug 日志。

**Tech Stack:** C++23、标准库、POSIX `fsync/rename`、doctest、CMake、nlohmann/json、libmosquitto。

## Global Constraints

- 文档、注释、提交说明和运行日志一律使用中文；技术标识符保留英文。
- 不引入 spdlog 或其他新依赖；`cns_control_test` 的输出契约不得改变。
- 任何级别都不输出逐帧 MAVLink 字段。
- `debug` 只记录 MQTT `Publish()` 成功接受的紧凑遥测 JSON；失败只记录 `warn`。
- 无效时间固定输出 `---------- --:--:--.---`，同时保留进程单调时间。
- 文件模式只保留一个文件，默认及现场值为 `1024 KiB`，不得生成历史文件。
- 滚动以完整日志行为边界，通过同目录临时文件、文件 `fsync`、原子替换和目录 `fsync` 提交。
- Logger 兼容主线程与 libmosquitto 回调线程，不启动后台线程或无界队列。

```cpp
namespace logging {
enum class Level { kDebug = 0, kInfo = 1, kWarn = 2, kError = 3 };
std::expected<Level, std::string> ParseLevel(std::string_view);
struct Options { Level minimum_level; std::filesystem::path file; std::size_t max_file_size_bytes; };
struct Clock {
  std::function<std::chrono::system_clock::time_point()> system_now;
  std::function<std::chrono::steady_clock::time_point()> steady_now;
};
class Logger {
 public:
  static std::expected<std::unique_ptr<Logger>, std::string> Create(
      Options, std::ostream&, std::ostream&, Clock = {});
  bool Enabled(Level) const;
  void Debug(std::string_view);
  void Info(std::string_view);
  void Warn(std::string_view);
  void Error(std::string_view);
 private:
  void Write(Level, std::string_view);
  mutable std::mutex mutex_;
};
void LogPublishedTelemetry(Logger&, std::string_view compact_json);
}  // namespace logging
```

- [ ] **Step 3: 实现有界文件输出**

增加私有接口：

```cpp
std::expected<void, std::string> InitializeFile();
bool AppendOrCompact(std::string line);
bool CompactAndAppend(std::string_view line);
void FallBackToStandardError(std::string_view reason);
```

创建时要求父目录存在、目标不是目录且可 append。未超限直接追加。超限时先把单条记录截断到容量内，再按以下算法选择保留起点：

```cpp
const std::size_t required_drop =
    old.size() + line.size() > maximum ? old.size() + line.size() - maximum : 0;
std::size_t keep_from = std::max(required_drop,
                                 std::min(old.size(), maximum / 4));
if (keep_from < old.size()) {
  const auto newline = old.find('\n', keep_from);
  keep_from = newline == std::string::npos ? old.size() : newline + 1;
}
const std::string replacement = old.substr(keep_from) + line;
```

同目录 `mkstemp`，完整 write、文件 `fsync`、`rename`、目录 `fsync`；失败清理临时文件。运行时失败只报告一次降级原因，后续走 stdout/stderr。
