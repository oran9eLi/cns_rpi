# 5G 链路延迟与丢包率上报实施计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 每 5 秒测量设备经 `usb0` 到 `112.124.52.232` 的 RTT，并在 MQTT 遥测中上报最近 12 次的平均延迟和丢包率。

**Architecture:** Python 5G 守护服务负责 ICMP 采样、滑动统计和快照写入；C++ 主程序只兼容读取两个可空字段并透传到 `telemetry.cellular_5g`。质量采样采用独立调度节拍，不改变既有双目标状态机、恢复策略和 STM32 `RPICELL`。

**Tech Stack:** Python 3 标准库、Linux `ping`、C++23、nlohmann/json、unittest、doctest、CMake/CTest。

## Global Constraints

- 所有探测必须使用 `ping -I usb0`，不得回退到 Wi-Fi。
- 业务质量目标固定为 `112.124.52.232`；`119.29.29.29` 仅保留现有连通性判定职责。
- 每 5 秒采样 1 次，最近 12 次组成滑动窗口。
- 延迟和丢包率保留 1 位小数；全失败时延迟为 `null`、丢包率为 `100.0`。
- 单次质量探测失败不写 `last_error`、不触发恢复、不改变 `RPICELL`。
- 文档和程序输出使用中文；Git 提交前缀保持英文。
