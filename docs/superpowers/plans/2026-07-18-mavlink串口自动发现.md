# MAVLink 串口自动发现实施计划

> **面向执行代理：** 必须使用 `superpowers:subagent-driven-development`（推荐）或 `superpowers:executing-plans`，逐任务执行本计划。每一步使用复选框跟踪。

**目标：** 让主程序和本地控制工具通过合法 MAVLink 帧自动发现 STM32 串口，并在启动缺失或运行中断线时持续等待、自动恢复。

**架构：** `uart/mavlink_port_discovery` 负责候选枚举、两端交替排序和单轮协议探测，成功结果保留首帧及已打开链路。`MavlinkLink` 用 `std::expected` 区分无帧与读写故障；主程序持有可空链路，断线后清理端点并回到发现状态，同时保留 MQTT 客户端。

**技术栈：** C++23、POSIX termios/PTY、官方 MAVLink C 头文件、doctest、CMake、systemd、Raspberry Pi ARM64。

## 全局约束

- 文档、代码注释、日志、诊断信息和提交信息一律使用中文。
- `serial.device="auto"` 是正式部署模式；明确路径只探测该路径。
- 自动模式枚举 `/dev/ttyUSB*` 和 `/dev/ttyACM*`，按尾号整数自然排序后执行最低、最高两端交替。
- USB路径、VID/PID、interface number 只用于枚举和诊断，不能确认 STM32 身份。
- 任意通过官方 MAVLink CRC 校验的完整帧都能确认串口；首帧必须保留并交给业务层。
- STM32 `sysid/compid` 仍只从固定频率 HEARTBEAT 学习，不能写死动态 `sysid`。
- 找不到串口时进程不退出；重复等待告警最多每10秒一条，并响应 `SIGINT`/`SIGTERM`。
- 连续10秒没有收到合法 MAVLink 帧时按链路失效处理；只有合法完整帧刷新计时。
- 运行中串口失效后 MQTT 保持连接，下行命令立即返回 `stm32_link_unavailable`。
- 不修改 STM32 固件、官方生成的 `src/mavlink/`、真实 `config/config.json` 或 RM500U 拨号逻辑。
- 单元测试不得依赖真实硬件；树莓派只做最终 ARM64 与插拔验收。

MavlinkLink::ReceiveMessage() {
  std::array<std::uint8_t, 256> buffer{};
  auto count = port_.Read(buffer);
  if (!count) return std::unexpected(count.error());
  return assembler_.Feed(
      std::span<const std::uint8_t>(buffer.data(), *count));
}

std::expected<void, UartError>
MavlinkLink::SendMessage(const mavlink_message_t& message) {
  std::array<std::uint8_t, MAVLINK_MAX_PACKET_LEN> buffer{};
  const std::uint16_t length =
      mavlink_msg_to_send_buffer(buffer.data(), &message);
  auto written = port_.Write(
      std::span<const std::uint8_t>(buffer.data(), length));
  if (!written) return std::unexpected(written.error());
  if (*written != length) return std::unexpected(UartError::kWriteError);
  return {};
}
```

    link.emplace(std::move(attempt.found->link));
    silence_watchdog.ObserveValidFrame(std::chrono::steady_clock::now());
    (*logger)->Info("已发现STM32 MAVLink串口: " + active_serial_device);
  } else {
    if (discovery_log_limiter.ShouldLog(std::chrono::steady_clock::now())) {
      (*logger)->Warn("尚未发现STM32 MAVLink串口，继续等待");
    }
    next_discovery = std::chrono::steady_clock::now() + kDiscoveryRetryInterval;
  }
}
```

把首帧和正常接收帧统一交给同一个局部处理函数，避免复制端点学习、ACK和解码逻辑。
该函数每处理一条合法完整帧就调用 `silence_watchdog.ObserveValidFrame(now)`；没有完整帧、
普通字节和CRC错误帧均不刷新。每轮检查 `silence_watchdog.Expired(now)`，达到10秒时先
记录“连续10秒未收到合法MAVLink帧”，再调用同一个 `mark_link_disconnected`，不能另写
一套清理分支。
任何 `ReceiveMessage()`/`SendMessage()` 错误调用统一断线处理：

```cpp
auto mark_link_disconnected = [&] {
  (*logger)->Error("STM32串口断开: " + active_serial_device);
  link.reset();
  active_serial_device.clear();
  pending_first_message.reset();
  stm32_endpoint.reset();
  silence_watchdog.Reset();
  next_discovery = std::chrono::steady_clock::now();
  if (control_transaction.HasPending()) {
    (void)control_transaction.HandleLocalFailure({
        .code = "stm32_link_unavailable",
        .message = "STM32串口链路不可用"});
  }
};
```

发送心跳和5G状态前要求 `link && rpi_system_id`。下行控制提交后先判断 `!link`，返回
`stm32_link_unavailable`；链路存在但尚未收到心跳时仍返回现有
`stm32_identity_unknown`。断线期间不要重置 `mqtt_client`、topic、注册状态或遥测状态。
链路为空且尚未到下一次扫描时间时，每轮最多 `sleep_for(100ms)`，避免空转占满CPU；
睡眠保持短周期，确保 `SIGINT`/`SIGTERM` 后能在100ms量级退出。
