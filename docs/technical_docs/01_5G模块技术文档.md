## 1. 模块

5G 模块为 Raspberry Pi 5 提供蜂窝 IP 网络接入，使树莓派在现场没有有线网络或 Wi-Fi 时仍可连接外部 IP 网络。本工程实装的器件是 **Quectel RM500U-CNV**，不是产品设计书建议的 RG660Qx，也不将其描述为 5G-A 模块。

模块通过 USB 与树莓派连接：USB 串口用于发送 AT 指令，NCM 网卡 `usb0` 用于承载 IP 数据。常驻 Python 服务负责首次拨号、链路监测、无线质量采集和分级恢复；`cns_rpi` C++ 主程序只读取运行时状态快照，不占用 AT 口。STM32 UART/MAVLink 链路不依赖 5G 服务是否在线。

正式接入链路：

```text
Quectel RM500U-CNV
  -> USB 复合设备（USB ID 2c7c:0900）
  -> /dev/ttyUSB*（按 USB interface 05 动态识别主 AT 口）
  -> scripts/cellular_dialup.py
       1. 查询/设置 PDP 上下文
       2. 激活 PDP
       3. 将 PDP 绑定到 NCM 网卡
       4. 强制经 usb0 探测双公网目标
       5. 采集 QENG/CSQ/COPS 和流量计数
       6. 断链后重拨，必要时软重启模块
  -> usb0（LOWER_UP）
  -> NetworkManager（DHCP/SLAAC 与路由）
  -> IP 网络
  -> /run/cns-rpi/cellular_status.json
  -> cns_rpi（MQTT cellular_5g + 原 RPICELL 在线位）
```

本文只覆盖上述 5G 接入链路，不展开运行在 IP 网络之上的 MQTT 遥测、服务器协议和 STM32 数据链路。

## 3. 硬件与总线

| 项目 | 实际配置 |
|---|---|
| 器件 | Quectel RM500U-CNV |
| 实测固件版本 | `RM500UCNVAAR03A14M2G` |
| 主机 | Raspberry Pi 5，Raspberry Pi OS 64-bit |
| 物理接口 | USB 复合设备 |
| USB ID | `2c7c:0900` |
| 主 AT 口 | USB interface number `05`；节点编号不固定 |
| AT 串口参数 | 115200 bit/s，pyserial 默认 8N1，读超时 1 s |
| 数据接口 | NCM 网卡 `usb0`，netcard index `0` |
| 当前 SIM | 中国移动物联卡，未启用 PIN 锁 |
| 当前 APN/PDP | `CMMTM5GDCDW.JS`，CID `1`，PDP 类型 `IPV4V6` |

RM500U-CNV 本次枚举出 5 个 `/dev/ttyUSB*` 端口。设备节点编号会随其他 USB 设备插拔而变化，因此程序不写死 `/dev/ttyUSB3`，而是对 `/dev/ttyUSB*` 执行 `udevadm info`，按 `ID_USB_INTERFACE_NUM=05` 识别主 AT 口。

实测 USB interface 映射如下，仅作为诊断参考：

| USB interface | 实测用途 |
|---|---|
| `02` | 未测试 |
| `03` | 诊断/日志口，不是 AT 口 |
| `04` | 可接收 AT，但不是主 AT 口 |
| `05` | 主 AT 指令口 |
| `06` | 未测试 |

## 4. 软件分层与依赖

| 层/组件 | 文件或系统服务 | 职责 |
|---|---|---|
| 配置 | `config/config.example.json` 的 `cellular` 节 | 提供拨号、探测、迟滞、质量采集和恢复参数 |
| 链路领域逻辑 | `scripts/cellular_link.py` | 配置校验、QENG/CSQ/COPS 解析、状态机、退避和快照模型 |
| 常驻守护进程 | `scripts/cellular_dialup.py` | 动态识别 AT 口、首次拨号、链路探测、质量采集、分级恢复和原子快照 |
| 进程管理 | `systemd/cellular-dialup.service` | 以 `dcdw` 用户常驻运行；进程异常退出后等待 15 s 重启 |
| 网卡配置 | NetworkManager | 在 `usb0` 出现载波后完成 DHCP/SLAAC 和路由配置 |
| 依赖安装 | `scripts/install_deps.sh` | 安装 `python3-serial` 等运行和构建依赖 |
| 自动化测试 | `tests/test_cellular_link.py`、`tests/test_cellular_dialup.py`、`tests/test_cellular_snapshot.cpp` | 验证纯逻辑、运行时适配器和 C++ 快照消费，不访问真实硬件 |

边界约束：

- 5G 服务是独立 Python 进程，不复用 C++ 的 `uart::SerialPort`；Python 测试由 CTest 统一调度。
- 脚本只读取 `config.json` 的 `cellular` 节，不读取 MQTT、STM32 串口或业务数据配置。
- 脚本不执行 DHCP、不修改路由表；这些职责属于 NetworkManager。
- 运行期断链和恢复由进程内部状态机处理；systemd 只处理进程异常退出。
- 该服务是 RM500U AT 口的唯一所有者；C++ 主程序只读取状态快照。
- `cns-rpi.service` 与 `cellular-dialup.service` 共用 `/run/cns-rpi`，两个单元均设置 `RuntimeDirectoryPreserve=yes`；任一服务单独停止或重启时不删除另一服务仍在使用的运行目录。

## 5. 数据结构与契约

本模块通过 `/run/cns-rpi/cellular_status.json` 向主程序提供状态。公开字段进入 `telemetry.cellular_5g`；内部 `diagnostics` 只用于保持 STM32 原有 `RPICELL` 位图语义。

### 5.1 配置契约

```json
"cellular": {
  "apn": "CMMTM5GDCDW.JS",
  "cid": 1,
  "usb_interface_number": "05",
  "at_port_wait_seconds": 30,
  "interface_name": "usb0",
  "probe_targets": ["112.124.52.232", "119.29.29.29"],
  "probe_interval_seconds": 10,
  "quality_probe_interval_seconds": 5,
  "offline_failure_threshold": 3,
  "online_success_threshold": 2,
  "signal_sample_interval_seconds": 30,
  "redial_attempts_before_reset": 3,
  "recovery_delay_seconds": 15,
  "recovery_delay_max_seconds": 300
}
```

| 字段 | 类型 | 当前示例值 | 说明 |
|---|---|---|---|
| `apn` | 字符串 | `CMMTM5GDCDW.JS` | 正式物联卡实机查询到的专用 APN |
| `cid` | 整数 | `1` | 用于上网的 PDP context ID |
| `usb_interface_number` | 字符串 | `05` | 主 AT 口的 USB interface number；保留前导零 |
| `at_port_wait_seconds` | 整数 | `30` | 等待 AT 口枚举的最长时间，单位为秒 |
| `interface_name` | 字符串 | `usb0` | NCM 数据接口名称 |
| `probe_targets` | 字符串数组 | 两个已验证公网 IP | 必须经 `usb0` 访问的连通性目标 |
| `probe_interval_seconds` | 整数 | `10` | 双目标链路检测周期 |
| `quality_probe_interval_seconds` | 整数 | `5` | 到 CNS 公网弹性 IP 的延迟与丢包采样周期 |
| `offline_failure_threshold` | 整数 | `3` | 双目标连续失败的离线阈值 |
| `online_success_threshold` | 整数 | `2` | 恢复后连续成功的上线阈值 |
| `signal_sample_interval_seconds` | 整数 | `30` | 无线质量采集周期 |
| `redial_attempts_before_reset` | 整数 | `3` | 软重启模块前的重拨次数 |
| `recovery_delay_seconds` / `recovery_delay_max_seconds` | 整数 | `15` / `300` | 恢复退避初值和上限 |

配置缺失、字段类型错误或文件不可读时，脚本打印错误并以非零状态退出。

### 5.2 AT 与状态契约

公开快照在既有字段之外增加 `latency_ms` 和 `packet_loss_percent`：前者为
最近 12 次业务入口探测中成功样本的平均 RTT，后者为失败样本占比。尚无探测
结果时两项均为 `null`；窗口内全部失败时分别为 `null` 和 `100.0`。这两个
字段只进入 MQTT 遥测，不改变 `RPICELL` 位图。

| 契约 | 含义 |
|---|---|
| AT 命令结束符 | `\r\n` |
| 单条 AT 命令超时 | 5 s |
| 成功终止行 | `OK` |
| 失败终止行 | `ERROR` 或 `+CME ERROR...` |
| `ONLINE` | 基础网络正常且两个公网目标均可达 |
| `DEGRADED` | 只有一个目标可达，或失败次数尚未达到离线阈值 |
| `OFFLINE` | 基础网络缺失，或双目标连续 3 轮失败 |
| `RECOVERING` | 正在重拨、等待重新联网或软重启模块 |
| `UNKNOWN` | 启动期、快照缺失或快照超过 30 s 未刷新 |

退出码 `0` 不等价于已经完成公网业务请求；IP 地址分配、DNS 和默认路由仍由 NetworkManager 负责。

## 6. 拨号 / 处理流程

1. systemd 在开机阶段启动 `scripts/cellular_dialup.py <config.json>`。
2. 脚本每 1 s 扫描一次 `/dev/ttyUSB*`，通过 `udevadm` 匹配配置的 USB interface number；单次扫描窗口超时后进入有界退避，不退出守护进程。
3. 以 115200 bit/s 打开主 AT 串口。
4. 发送 `AT+CGDCONT?`，查询目标 CID 对应的 PDP 上下文。
5. 若目标 CID 不存在或 APN 与配置不一致，发送：

   ```text
   AT+CGDCONT=<cid>,"IPV4V6","<apn>"
   ```

6. 发送 `AT+CGACT=1,<cid>` 激活 PDP。若命令返回错误，再用 `AT+CGACT?` 查询实际状态；目标 CID 已激活时继续，否则失败退出。
7. 发送 `AT+QNETDEVCTL=<cid>,1,0`，将 PDP 上下文绑定到 NCM 网卡 0。
8. 最多等待 10 s，轮询 `ip link show usb0`；看到 `LOWER_UP` 后进入常驻监测。
9. NetworkManager 检测到 `usb0` 载波后自动完成 IPv4 DHCP、IPv6 SLAAC 和路由配置。
10. 每 10 s 强制经 `usb0` 探测两个公网目标；另每 5 s 探测 CNS 公网弹性 IP，按最近 12 次结果计算平均延迟与丢包率；每 30 s 查询无线质量并刷新快照。
11. 双目标连续 3 轮失败后重拨；三次重拨失败后执行 `AT+CFUN=0,1`，随后按上限 300 s 退避。

本流程是在 `AT+QCFG="usbnet"` 返回模式 `5`（NCM）的实机环境中验证的，不应直接套用于 MBIM、QMI 等其他 USB 网络模式。

## 7. 接口清单

### 7.1 脚本入口

```bash
python3 scripts/cellular_dialup.py config/config.json
```

参数必须是包含 `cellular` 节的 JSON 配置文件路径。正常情况下进程常驻，收到终止信号后返回 `0`；配置读取或校验失败返回 `1`；参数数量错误返回 `2`。拨号失败由进程内部恢复状态机处理，不会因单次失败退出。

### 7.2 主要内部接口

| 接口 | 职责 |
|---|---|
| `load_cellular_config(path)` | 读取并转换 `cellular` 配置 |
| `find_at_port(interface_number, wait_seconds)` | 动态扫描并返回匹配的 AT 设备节点 |
| `send_at_command(ser, command, timeout)` | 发送一条 AT 命令并等待终止行 |
| `parse_cgdcont_apn(lines, cid)` | 从 `AT+CGDCONT?` 响应解析指定 CID 的 APN |
| `parse_cgact_active(lines, cid)` | 从 `AT+CGACT?` 响应判断指定 CID 是否激活 |
| `wait_for_carrier(iface, timeout_seconds)` | 等待 Linux 网卡出现 `LOWER_UP` |
| `dial_up(config)` | 编排一次完整拨号流程，返回布尔结果 |

### 7.3 systemd 入口

`systemd/cellular-dialup.service` 当前按真机部署路径执行：

```text
/usr/bin/python3 /home/dcdw/cns_rpi/scripts/cellular_dialup.py /var/lib/cns-rpi/config.json
```

若部署用户或仓库路径变化，安装 service 前必须同步修改 `ExecStart`。

## 9. 异常与恢复

| 异常 | 当前处理 |
|---|---|
| 配置文件不可读、缺字段或类型错误 | 打印错误，非零退出 |
| AT 口未在时限内枚举 | 记录超时与目标 interface number，进入退避后继续扫描 |
| `udevadm` 查询单个节点失败 | 跳过该节点，继续扫描 |
| 主 AT 口打开失败 | 记录设备节点和异常，当次拨号失败并进入退避 |
| AT 命令超时、返回 `ERROR` / `+CME ERROR` | 当次拨号失败；`CGACT` 例外地先查询实际激活状态 |
| `usb0` 未在 10 s 内出现 `LOWER_UP` | 记录超时，当次拨号失败并进入退避 |
| 单次重拨失败 | 进程内部状态机按 15 s 起始、300 s 上限退避 |
| 连续三次重拨失败 | 执行 `AT+CFUN=0,1`，重新发现动态 AT 口后继续拨号 |
| 模块拔出 | 写入 `present=false/OFFLINE` 并持续重新发现；插回后自动拨号 |
| 快照缺失、损坏或过期 | C++ 主程序输出 `UNKNOWN`，继续发布其他遥测 |
| 主服务或 5G 服务单独停止 | systemd 保留共享运行目录，不中断另一服务的快照读写 |
| 模块状态异常需要平台报警 | 状态已进入 MQTT 遥测；服务端告警规则不在本轮实现范围 |

5G 恢复不会重启树莓派或终止 `cns_rpi`。STM32 仍只接收原有 `RPICELL` 在线位，不需要固件改动。

## 10. 调试与验证

### 10.1 已完成的实机验证

实机记录见 `docs/5G模块拨号验证.md`。在普通中国移动手机 SIM、APN `cmnet` 条件下已确认：

- SIM 状态 `READY`，模块已附着分组域。
- `AT+QNWINFO` 返回 `NR5G-SA`，实测注册到 5G SA 网络。
- `AT+CGACT=1,1` 可激活 CID 1；仅激活 PDP 时 `usb0` 仍可能无载波。
- `AT+QNETDEVCTL=1,1,0` 后 `usb0` 进入 `LOWER_UP`。
- NetworkManager 可自动获取 IPv4 地址、IPv6 地址和默认路由。
- 通过 `usb0` 的 DNS 与公网连通性已经人工验证。

上述结果证明当前模块、当前固件、当前 SIM 和 NCM 配置下的拨号链路可用，不代表任何环境下的固定吞吐率或时延保证。

2026-07-21 更换正式中国移动物联卡后，实机查询确认：

- `AT+CPIN?` 返回 `READY`，`AT+COPS?` 返回 `China Mobile`。
- CID 1 已激活，`AT+CGCONTRDP=1` 返回网络分配 APN `CMMTM5GDCDW.JS`。
- 仓库默认 APN 据此改为 `CMMTM5GDCDW.JS`；普通手机卡的 `cmnet` 仅作为历史验证记录保留。
- 拨号服务已自动改写 PDP 上下文并成功退出，`usb0` 获得 IPv4 地址和默认路由。
- 指定 `usb0` 的公网 ICMP 验证三次全部成功。

2026-07-22 对 RM500U 和物联网卡进一步验证：

- `AT+QENG="servingcell"` 返回 `NR5G-SA` 及 RSRP、RSRQ、SINR；`AT+CSQ` 可补充 RSSI。
- 当前固件不支持 `AT+QCSQ`，代码不使用该命令。
- 强制经 `usb0` 访问 `112.124.52.232` 和 `119.29.29.29` 均为 3/3 成功；前者的 1883/TCP 可连接。
- `223.5.5.5` 虽可达但本轮丢包 33%，未选作主要探测目标。

2026-07-22 完成常驻服务和 USB 拔插恢复终验：

- ARM64 树莓派完成全量构建，CTest 30/30 通过；Python 链路单元测试 30 项通过。
- 正常在线时，`cellular-dialup.service` 为 `active/running`、`NRestarts=0`，快照为 `ONLINE`；实测接入制式为 `NR5G-SA`、运营商为 `China Mobile`。
- 拔出 RM500U 后，同一守护进程进入 `RECOVERING`并持续扫描 AT 口；插回后动态识别 `/dev/ttyUSB4`，在退避窗口到期后自动拨号。
